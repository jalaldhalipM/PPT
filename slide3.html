Components: Functional vs Class

Understanding the evolution of React components and why Functions + Hooks are the modern standard.

Recommended

Functional Component

Simpler syntax with less boilerplate

Uses Hooks (useState, useEffect) for logic

Easier to read, test, and maintain 

function Greeting({ name }) {
  // Logic is simpler & direct
  
  return (
    <div className="card">
      <h2>Hello, {name}</h2>
    </div>
  );
}
Legacy Support

Class Component

Requires extending React.Component

Uses this keyword (can be confusing)

Complex lifecycle methods (componentDidMount) 

class Greeting extends React.Component {
  render() {
    return (
      <div className="card">
        <h2>
          Hello, {this.props.name}
        </h2>
      </div>
    );
  }
}
